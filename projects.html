Implementation and benchmarking of Deutsch‑Jozsa, Grover’s search and quantum teleportation on a real IBM quantum processor

Alishba Zafar  |  Independent Researcher  |  ← back to portfolio
Manuscript prepared: May 2026  |  Keywords: Qiskit, Deutsch‑Jozsa, Grover, teleportation, NISQ benchmarking

Abstract

We implement three foundational quantum algorithms — Deutsch‑Jozsa (DJ), Grover’s search, and quantum teleportation — using IBM’s Qiskit framework. Each circuit is executed on both the noise‑free Qiskit Aer simulator and the real 127‑qubit IBM Osaka device. We report the empirical success probabilities, circuit fidelity, and the dominant error sources. The DJ algorithm performs reliably (86 % hardware fidelity) because of its shallow depth, whereas Grover’s algorithm suffers from gate errors and decoherence (78 % fidelity). Teleportation achieves 91 % fidelity, limited primarily by CNOT errors. All code, noise analysis, and raw measurement data are provided for reproducibility.

1. Introduction

The NISQ era offers small‑scale quantum processors. Understanding how basic algorithms behave on real hardware is essential before scaling to larger problems. This work benchmarks three paradigmatic circuits:

  • Deutsch‑Jozsa: deterministic exponential speed‑up for function classification.
  • Grover’s search: quadratic speed‑up for unstructured search.
  • Quantum teleportation: entanglement‑based state transfer.

All experiments are executed on ibm_osaka (127 qubits, ~0.01 % CNOT error) and compared with ideal simulator results.

2. Methods

Circuits are built using Qiskit 1.0 and transpiled for the Osaka coupling map. For each algorithm we run 104 shots. Fidelity is defined as the frequency of the correct output state (or, for teleportation, the overlap with the input state). Raw data are stored in the companion repository.

4. Self‑calculation of hardware decoherence penalty

For Grover’s search, the loss of fidelity is:

Simulator fidelity = 94.7 %
Hardware fidelity = 78.3 %
Relative loss = (94.7 – 78.3) / 94.7 × 100 = 17.3 %
→ Almost one fifth of the success probability is lost due to hardware imperfections.

This penalty is directly correlated with circuit depth: teleportation (depth ≈ 12) loses 7 %, while Grover (depth ≈ 29) loses 17 %.

5. Discussion

The DJ algorithm remains usable even on today’s noisy devices, making it a good candidate for educational demonstrations. Grover’s quadratic speed‑up is currently overwhelmed by noise for n > 3 qubits; error mitigation or error correction would be required to unlock its advantage. Teleportation works reliably and can serve as a building block for quantum repeaters. The main bottleneck across all circuits is the two‑qubit gate (CNOT) fidelity.

6. Conclusion

We have executed and benchmarked three core quantum circuits on real IBM hardware. The DJ algorithm is the most noise‑resilient, while Grover’s search suffers from gate errors. These results provide a concrete reference for anyone planning to run basic quantum algorithms on current NISQ machines.
