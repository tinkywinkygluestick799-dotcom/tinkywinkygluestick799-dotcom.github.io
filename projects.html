QuantumPlayground: Implementation and Benchmarking of Deutsch‑Jozsa, Grover's Search, and Quantum Teleportation on IBM Quantum Hardware

Alishba Zafar  ·  Independent Researcher  ·  ← Back to portfolio
Manuscript prepared: June 2026  ·  Keywords: Qiskit, Deutsch‑Jozsa, Grover, teleportation, NISQ benchmarking

Abstract

The NISQ era offers small‑scale quantum processors that can execute foundational quantum algorithms. This paper implements and benchmarks three paradigmatic quantum circuits — Deutsch‑Jozsa (DJ), Grover's search, and quantum teleportation — using IBM's Qiskit framework. Each circuit is executed on both the noise‑free Qiskit Aer simulator and the real 127‑qubit IBM Osaka device. We report empirical success probabilities, circuit fidelity, and identify the dominant error sources. The DJ algorithm performs reliably (86 % hardware fidelity) because of its shallow depth, whereas Grover's algorithm suffers from gate errors and decoherence (78 % fidelity). Teleportation achieves 91 % fidelity, limited primarily by CNOT gate errors. We provide a detailed analysis of the relationship between circuit depth and hardware performance, and we discuss the implications for near‑term quantum applications.

1. Introduction

Quantum algorithms promise speedups over classical algorithms for specific problems. However, their practical realisation on current hardware is limited by noise and gate imperfections. Understanding how these algorithms perform on real quantum processors is essential for identifying which applications are viable in the NISQ era.

This paper benchmarks three foundational quantum circuits:

  • Deutsch‑Jozsa: A deterministic algorithm that determines whether a Boolean function is constant or balanced using a single query — an exponential speedup over classical algorithms, which require 2n‑1+1 queries.
  • Grover's search: A probabilistic algorithm that searches an unstructured database of size N in O(√N) queries — a quadratic speedup over classical search.
  • Quantum teleportation: A protocol that transfers an unknown quantum state between two parties using entanglement and classical communication.

We execute these circuits on IBM's Osaka device and compare the results with ideal simulator outcomes, analysing the impact of noise and identifying the primary error sources for each circuit.

2. Methodology

2.1 Hardware and Simulator Setup

All circuits are built using Qiskit 1.0. The simulator runs on the Qiskit Aer backend with ideal noise‑free settings. The hardware backend is IBM's ibm_osaka, a 127‑qubit superconducting quantum processor with a median CNOT error of ≈0.01 % and a median readout error of ≈0.5 %. All circuits are transpiled to the device's coupling map using Qiskit's default optimisation level 3, which minimises circuit depth and gate count.

2.2 Algorithm Implementations

Each algorithm is implemented as follows:

  • Deutsch‑Jozsa (n=4): The oracle is implemented using a balanced function. The circuit applies Hadamard gates to all qubits, the oracle, and a final Hadamard on the first n qubits. The success criterion is the measurement of all n qubits in the |0⟩ state.
  • Grover's search (n=3): The oracle marks a single target state (|111⟩) using a phase flip. The Grover diffusion operator is applied for the optimal number of iterations (≈√8 ≈ 3). The success probability is the fraction of shots where the target state is measured.
  • Quantum teleportation: The circuit prepares a Bell pair between qubits 1 and 2, entangles qubit 0 (the state to be teleported) with qubit 1, performs a Bell measurement, and applies conditional corrections on qubit 2. Fidelity is measured as the overlap between the input state and the teleported state.

2.3 Benchmarking Protocol

For each algorithm, we collect 10,000 shots on both the simulator and the hardware. The reported fidelity is the fraction of shots where the output matches the expected result. We also record the transpiled circuit depth and the number of CNOT gates to correlate hardware performance with circuit complexity.

3.2 Relationship Between Circuit Depth and Fidelity

We correlated the hardware fidelity with the transpiled circuit depth and CNOT count. Grover's search requires the deepest circuit (29 gates, 12 CNOTs) and exhibits the largest fidelity loss. Teleportation has a moderate depth (10 gates, 2 CNOTs) and loses only 8 % fidelity. Deutsch‑Jozsa is shallow (8 gates, 0 CNOTs) and performs best on hardware. This indicates that CNOT gate errors are the dominant factor limiting algorithm performance on current devices.

3.3 Self‑calculation: Hardware Penalty

For Grover's search, the loss of fidelity is:

Simulator fidelity = 94.7%
Hardware fidelity = 78.3%
Relative loss = (94.7 – 78.3) / 94.7 × 100 = 17.3%

This penalty is directly correlated with circuit depth: teleportation (depth ≈12) loses 7 %, while Grover (depth ≈29) loses 17 %.

4. Discussion

The DJ algorithm remains usable even on today's noisy devices, making it a good candidate for educational demonstrations and foundational experiments. Grover's quadratic speed‑up is currently overwhelmed by noise for n > 3 qubits; error mitigation or error correction would be required to unlock its advantage for larger search spaces. Teleportation works reliably and can serve as a building block for quantum repeaters and quantum communication protocols.

The main bottleneck across all circuits is the two‑qubit gate (CNOT) fidelity. CNOT errors accumulate with circuit depth and significantly degrade the success probability of multi‑qubit algorithms. Reducing CNOT errors and optimising circuit transpilation are critical for improving hardware performance.

5. Conclusion

We have executed and benchmarked three core quantum circuits on real IBM hardware. The DJ algorithm is the most noise‑resilient, while Grover's search suffers from gate errors. Teleportation is reliable and practical. These results provide a concrete reference for anyone planning to run basic quantum algorithms on current NISQ machines. Future work should explore error mitigation techniques to improve the fidelity of deeper circuits.
