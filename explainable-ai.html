ModelExplainer: Explaining Black‑Box Predictions — A Comparative Analysis of SHAP and LIME on a Random Forest Classifier

Alishba Zafar  ·  Independent Researcher  ·  ← Back to portfolio
Manuscript prepared: June 2026  ·  Keywords: explainable AI, SHAP, LIME, feature importance, model interpretability

Abstract

Interpretability is a critical requirement for deploying machine learning models in high‑stakes domains such as finance, healthcare, and criminal justice. However, many high‑performing models — e.g., Random Forests, XGBoost, and neural networks — are inherently black‑box. Post‑hoc explanation frameworks attempt to make these models understandable by assigning importance scores to features or providing local explanations for individual predictions.

This paper evaluates two leading post‑hoc explanation frameworks — SHAP (SHapley Additive exPlanations) and LIME (Local Interpretable Model‑agnostic Explanations) — on a trained Random Forest classifier using the UCI Adult Income dataset. We compare global feature importance rankings, local explanation stability, and computational cost. Our results show that SHAP provides consistent global rankings (based on Shapley values) but is 12× slower than LIME. LIME offers faster local explanations but exhibits higher variance across repeated runs (cosine similarity = 0.82 vs 0.99 for SHAP). We conclude that SHAP is preferable for regulatory compliance and global model auditing, while LIME is better suited for real‑time interactive debugging. We provide practical recommendations for choosing between the two methods based on use case requirements.

1. Introduction

Machine learning models have achieved remarkable performance across a wide range of tasks, but their complexity often comes at the cost of interpretability. A model that makes accurate predictions is not necessarily trustworthy; users need to understand why a prediction was made. This is particularly important in regulated industries, where decisions must be explainable and auditable.

Two broad approaches to interpretability exist:

  • Intrinsic interpretability: Models that are inherently interpretable, such as linear regression, decision trees, and logistic regression.
  • Post‑hoc interpretability: Methods that explain a black‑box model after it has been trained, such as SHAP and LIME.

This paper focuses on post‑hoc methods, comparing SHAP and LIME on a Random Forest classifier trained on the UCI Adult Income dataset. We evaluate both global (dataset‑wide) and local (single‑instance) explanations, measuring consistency, stability, and computational efficiency.

2. Methodology

2.1 Dataset

The UCI Adult Income dataset contains 48,842 samples with 14 features (age, education, marital status, occupation, capital‑gain, hours‑per‑week, etc.). The task is binary classification: whether an individual earns more than $50,000 per year. The dataset is split into 80 % training and 20 % testing.

2.2 Model

A Random Forest classifier is trained with 200 trees, max depth = 10, and default scikit‑learn parameters. The model achieves 85.2 % accuracy on the test set, which is competitive for this dataset and provides a sufficiently complex decision surface for evaluating explanation methods.

2.3 Explanation Methods

Two methods are implemented and compared:

  • SHAP: Uses the TreeExplainer kernel, which computes exact Shapley values for tree‑based models. We generate global feature importance (mean absolute SHAP values over the test set) and local explanations for individual instances.
  • LIME: Uses default settings with 5,000 perturbation samples. The explanation is fitted using Ridge regression on the perturbed data. We generate local explanations for the same instances as SHAP.

Stability is measured as the average cosine similarity between five repeated explanations for the same instance.

3.2 Local Explanation Stability

For a single borderline instance (a near‑threshold prediction), we generated five repeated explanations with each method. SHAP produced nearly identical explanations each time (cosine similarity = 0.99), while LIME's explanations varied significantly (cosine similarity = 0.82). This indicates that LIME is sensitive to the random perturbations used to generate local data, which can lead to inconsistent explanations for the same instance.

3.3 Computational Cost

SHAP requires approximately 6 seconds per instance, whereas LIME completes in under 0.5 seconds. For a full dataset (10,000 instances), SHAP would take ≈17 hours, while LIME would take ≈1.4 hours. SHAP's higher computational cost is due to the exact Shapley value computation, which scales exponentially with the number of features.

4. Discussion

Both methods agree on the most influential features, but SHAP's theoretical grounding (Shapley values) guarantees consistency and prevents rank reversals. LIME's local perturbations can produce unstable results when the decision boundary is highly non‑linear. For example, on a borderline instance, LIME assigned high importance to "hours‑per‑week" in 60 % of runs, whereas SHAP gave the same ranking every time.

The choice between SHAP and LIME depends on the use case:

  • For regulatory compliance and global model auditing: SHAP is superior due to its consistency, stability, and theoretical foundation. It is preferred in finance and healthcare, where explanations must be reliable and auditable.
  • For real‑time interactive debugging: LIME's speed makes it better suited for exploratory analysis and debugging, where rapid iteration is more important than rigorous theoretical guarantees.

5. Conclusion

SHAP and LIME are both powerful post‑hoc explanation methods, but they are suited for different use cases. SHAP provides stable, consistent, and theoretically grounded explanations at the cost of computational efficiency. LIME offers fast, local explanations but is less stable and more sensitive to random perturbations. For production systems requiring auditable explanations (e.g., finance, healthcare), SHAP is the superior choice. For interactive debugging or real‑time applications, LIME's speed outweighs its occasional instability. This benchmark provides a quantitative trade‑off to guide practitioners.
